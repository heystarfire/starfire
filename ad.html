Understanding Active Directory: Key Concepts and Terminology

Active Directory (AD) is a directory service developed by Microsoft for Windows domain networks. It provides a centralized way to manage and organize network resources, including users, computers, and other device

Active Directory Domain Services (AD DS) manages domains and provides a directory service for storing information about the domain's objects, including users, computers, groups, and organizational units (OUs)

Key Concepts in Active Directory:

Domain Controller (DC):Servers that host the Active Directory database and provide authentication and authorization services to users and computers within the domain.

Forest: A forest is the topmost logical container in an Active Directory configuration that contains one or more domains. A forest represents a single instance of the Active Directory directory service.

Domain: Within a forest, a domain represents a logical grouping of objects, such as users, computers, and groups, that share the same directory database.

Organizational Units (OUs): An OU is a container within a domain that can hold users, groups, computers, and even other OUs. OUs help organize objects within a domain and can also be used to apply Group Policy settings.

Users / User Accounts: In Active Directory (AD), users are objects that represent individuals or entities needing access to network resources. A user object stores information such as username, password, contact details, and group memberships, which are used for authentication and authorization.

User accounts in AD are stored in the default Users container. This container is not an Organizational Unit (OU), so Group Policy Objects (GPOs) cannot be linked directly to it.

In AD, there are two main types of user accounts:

  • Local Accounts: These accounts are stored on individual systems and are specific to each computer. Examples include Administrator, Guest, SYSTEM, Network Service, and Local Service accounts.
  • Domain User Accounts: These accounts are managed centrally in a domain and allow users to access resources across the entire domain

Computer Accounts: Computer accounts are identified by the $ symbol at the end of their name (e.g., COMPUTERNAME$). These accounts represent computers that are joined to the domain.

Example Path in Active Directory

OU Structure: OU=Computers,OU=test,DC=vb,DC=local

  • This path represents an Organizational Unit named "Computers" inside another OU named "test", under the domain "vb.local".

User Path: CN=starfire,OU=Users,DC=vb,DC=local

  • This path represents a user account named "starfire" in the "Users" OU under the domain "vb.local".

Groups: Groups are used to collect user accounts, computer accounts, and other groups into manageable units. Groups can be added as members of other groups (nested groups), which helps organize and simplify permissions management.

Group Membership Example:

If we create a group named "Cat Group" and add it to another group called "Remote Management Group," all members of the "Cat Group" will automatically inherit the permissions assigned to the "Remote Management Group." If we add a user named Robin to the "Cat Group," Robin will be a member of both the "Cat Group" and the "Remote Management Group."

Group Types and Scopes

  • Security Groups: Used for assigning permissions and access controls to resources.
  • Distribution Groups: Used for email distribution; cannot be used for assigning permissions.

Group Scopes

  1. Domain Local Group
    • Used to assign permissions to resources within the domain where the group is created.
    • Can contain users, global groups, and universal groups from any domain.
  1. Global Group
    • Typically used to organize users within the same domain.
    • Can only contain users and groups from the domain where it is created but can be granted access to resources in any domain within the forest.
  1. Universal Group
    • Used to assign permissions to resources across multiple domains within a forest.
    • Can contain users, global groups, and other universal groups from any domain within the forest.
    • Stored in the Global Catalog.

Group Policy (GP): Group Policy is a feature that allows administrators to define settings that control the working environment of user accounts and computer accounts. Group Policies can be applied to domains, sites, and OUs, but not directly to individual users or groups. You can apply GPOs only to the root of a domain or an OU

Hence Group Policies can be applied only to the root of a domain or to an OU, not directly to individual users or groups.

Examples of GPO Uses

Password Policies:

  • Create separate GPOs to establish different password policies for service accounts, admin accounts, and standard user accounts.

Removable Media Control:

  • Prevent the use of USB ports and other removable media devices to enhance security.

Screensaver Enforcement:

  • Enforce a screensaver with a password to ensure that unattended machines are locked.

Application Restrictions:

  • Restrict access to applications such as cmd.exe and PowerShell for standard users to minimize security risks.

Audit and Logging Policies:

  • Enforce policies for auditing and logging to track system and user activities.

Software Deployment:

  • Deploy software across the domain, ensuring that all users have the necessary applications installed.

Software Installation Control:

  • Block users from installing unauthorized software to maintain control over the software environment.

Logon Banner:

  • Display a logon banner with legal or compliance messages whenever a user logs into the system.

LM Hash Usage:

  • Disallow the use of LM (LAN Manager) hashes in the domain to enhance password security.

Script Execution:

  • Run scripts at startup/shutdown or user logon/logoff to perform specific tasks or configurations.

Active Directory Terminology

Object: Any resource within AD, such as users, computers, printers, or domain controllers.

Attributes: Characteristics of an object. For example, a computer object might have attributes like hostname and DNS name.

Schema: Defines the types of objects and their attributes within AD. It acts as a blueprint for the directory.

Domain: A logical grouping of objects. Each domain operates independently but can be linked through trusts.

Forest: The top-level container, encompassing one or more domains.

Tree: A collection of domains within a forest, starting from a root domain.

Global Unique Identifier (GUID): A unique 128-bit value assigned to each AD object.

Security Principal: An entity that the OS can authenticate, such as users or computer accounts.

Security Identifier (SID): A unique identifier assigned to security principals.

Distinguished Name (DN): The full path to an object in AD.

Relative Distinguished Name (RDN): A component of the DN identifying the object uniquely at the current level.

sAMAccountName: The user logon name.

Global Catalog: A domain controller that holds copies of all objects in the forest and allows for searching across domains.

Read-Only Domain Controller (RODC): A domain controller with a read-only AD database.

Replication: The process of synchronizing AD data between domain controllers.

Service Principal Name (SPN): Identifies a service instance for Kerberos authentication.

Access Control Entries (ACEs): Part of an ACL, detailing access rights for users or groups.

Group Policy Object (GPO): Collections of policy settings applied across domains.

Access Control List (ACL): An ordered list of ACEs applied to an object.

Discretionary Access Control List (DACL): Defines access rights for security principals.

Fully Qualified Domain Name (FQDN): The complete domain name for a specific host.

Tombstone: Holds information about deleted AD objects for a set period.

SYSVOL: A shared folder containing domain-wide public files like GPOs.

AdminSDHolder: Manages ACLs for privileged groups.

Active Directory Users and Computers (ADUC): A GUI tool for managing AD objects.

ADSI Edit: A tool for advanced AD object management.

sIDHistory: Stores previous SIDs for migrated objects.

NTDS.DIT: The database file containing AD data, including user password hashes.
